# Telegram-бот для изучения «Капитанской дочки»

Бот помогает проходить произведение по смысловым блокам. Он отправляет текст частями, задаёт вопросы по одному, принимает свободный ответ пользователя, сохраняет текущий прогресс и даёт короткие живые отклики через OpenAI API.

## Структура проекта

```text
bot.py
content/captains_daughter.json
services/message_sender.py
services/progress.py
services/llm_response.py
handlers/start.py
handlers/story.py
requirements.txt
.env.example
README.md
```

## Установка

1. Создай и активируй виртуальное окружение.
2. Установи зависимости:

```bash
pip install -r requirements.txt
```

3. Создай `.env` на основе `.env.example` и укажи токен бота и ключ OpenAI:

```env
BOT_TOKEN=your_telegram_bot_token_here
OPENAI_API_KEY=your_openai_api_key_here
```

## Запуск

```bash
python bot.py
```

## Команды бота

- `/start` — начать или продолжить курс
- `/help` — показать помощь
- `/progress` — показать текущий блок
- `/restart` — начать курс заново

## Как работает прогресс

Прогресс сохраняется в `data/progress.json`. Для каждого пользователя хранится:

- `user_id`
- `current_block`

## Как работают ответы OpenAI

После ответа ученика бот отправляет в OpenAI:

- название текущего блока
- текст блока
- текущий вопрос
- ответ пользователя

Перед генерацией бот определяет тип ответа ученика: `dont_know`, `very_short`, `weak` или `strong`. Для большинства осмысленных ответов приоритет у OpenAI: бот старается ответить адресно именно на мысль ученика. Fallback используется только для пустых, бессмысленных и явных ответов вроде `не знаю`, а также если API недоступен.

Модель получает только контекст текущего раздела и отвечает как спокойный, живой преподаватель литературы: коротко поддерживает мысль ученика и мягко подсвечивает важный смысл. Дополнительно бот проверяет ответы на слишком общие шаблоны и повторы недавних реплик, чтобы ответы были более живыми и менее однообразными.

Ключ OpenAI нужно указать в `.env`:

```env
OPENAI_API_KEY=your_openai_api_key_here
```
